# 🚀 HydraLLM 코드 리뷰 및 개선 제안

[한국어](IMPROVEMENTS.ko.md) | [English](IMPROVEMENTS.md)

이 문서는 HydraLLM의 종합적인 코드 리뷰를 바탕으로 현재의 강점, 약점 및 향후 개선 사항을 정리한 것입니다.

---

## 🏗 아키텍처 및 디자인

### 강점
- **Clean Architecture**: 핵심 도메인 모델, 비즈니스 로직(서비스), 인프라(어댑터) 간의 명확한 분리가 이루어져 있습니다.
- **인터페이스 기반**: `ILLMProvider`, `ISessionManager` 등을 통한 결합도 완화로 확장이 용이합니다.
- **상태 비저장 게이트웨이**: 핵심 라우팅 로직이 상태를 최소화하여 수평적 확장이 가능한 구조입니다.

### 개선 제안
- **게이트웨이 거대화 (Monolithic Gateway)**: `Gateway.process_request`가 세션 로드, 메시지 추출, 웹 인텐트 분석, 페칭, 검색, 컨텍스트 주입 및 라우팅을 모두 처리하고 있습니다.
  - *권장 사항*: 웹 관련 로직(탐지 -> 페칭 -> 주입)을 전담하는 `WebContextService`로 분리하는 것이 좋습니다.
- **의존성 관리**: 구성 요소 간의 순환 참조를 방지하고 테스트를 용이하게 하기 위해 의존성 주입 방식을 더 표준화해야 합니다.

---

## ⚡ 성능 및 확장성

### 강점
- **비동기 우선 (Async First)**: 대부분의 네트워크 작업과 스크래퍼 작업이 비동기로 구현되어 있습니다.
- **SQLite WAL 모드**: 동시성 높은 시나리오에서도 효율적인 로컬 영속성을 보장합니다.
- **컨텍스트 압축**: LLMLingua-2 통합을 통해 대화 기록이 길어져도 성능 저하를 방지합니다.

### 개선 제안
- **순차적 키 검증**: `probe_all_keys`가 여러 제공자에 대해 순차적으로 실행됩니다.
  - *권장 사항*: `asyncio.gather`를 사용하여 여러 제공자와 키를 병렬로 검증하도록 개선해야 합니다.
- **SQLite 오버헤드**: `asyncio.to_thread`로 이벤트 루프 블로킹은 막았으나, 메시지 파트의 잦은 JSON 직렬화/역직렬화 과정에서 최적화의 여지가 있습니다.

## 🔒 보안 및 안전성

### 강점
- **SSRF 방지**: 웹 스크래퍼가 URL을 검증하여 내부 네트워크 스캔을 차단합니다.
- **검증 (Validation)**: 모든 API 요청에 엄격한 Pydantic 모델을 사용합니다.
- **면책 조항 및 라이선스**: 실험용 도구로서의 법적 경계를 명확히 했습니다.

### 개선 제안
- **관리자 API 인증**: 현재 많은 관리자 엔드포인트가 기본적으로 엄격한 인증이 부족합니다.
  - *권장 사항*: `admin_api_key` 확인을 더 강화하거나, 운영 환경 이동 시 표준 OAuth/JWT와 통합해야 합니다.
- **키 저장 방식**: API 키가 `.env` 파일에 평문으로 저장됩니다.
  - *권장 사항*: Vault나 AWS Secrets Manager와 같은 외부 보안 관리 도구 지원이 필요합니다.

---

## 🛠 신뢰성 및 유지보수성

### 강점
- **견고한 폴백 (Fallback)**: 다단계 폴백(제공자 -> 제공자 -> 로컬 에이전트)을 통해 무료 티어에서도 99.9% 가용성을 확보합니다.
- **상관관계 ID (Correlation ID)**: Request ID 시스템을 통해 엔드 투 엔드 로그 추적이 가능합니다.
- **상세한 트러블슈팅**: 공통 실패 패턴에 대한 가이드가 잘 구축되어 있습니다.

### 개선 제안
- **프롬프트 주입의 취약성**: 문자열 치환이나 특정 인덱스에 의존하는 주입 방식은 모델 업데이트나 복잡한 사용자 입력에 의해 깨질 수 있습니다.
  - *권장 사항*: 주입된 컨텍스트를 위해 더 견고한 XML 스타일 태그나 특수 시스템 역할 마커를 사용해야 합니다.
- **테스트 커버리지**: 61개의 테스트가 존재하지만, 모든 제공자가 동시에 실패하는 극단적인 시나리오에 대한 통합 테스트 보강이 필요합니다.
