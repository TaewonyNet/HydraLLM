# HydraLLM 기여 가이드라인

먼저 기여를 위해 시간을 내주셔서 감사합니다! 🎉

다음은 HydraLLM에 기여하기 위한 일련의 지침입니다. 이것들은 규칙이라기보다 지침에 가깝습니다. 최선의 판단을 내리고, 풀 리퀘스트를 통해 이 문서의 변경 사항을 자유롭게 제안해 주세요.

## 어떻게 기여할 수 있나요?

### 버그 리포트

- 명확하고 설명적인 제목을 사용하세요.
- 문제를 재현할 수 있는 정확한 단계를 설명하세요.
- 예상했던 동작과 실제 발생한 동작이 무엇인지, 그리고 왜 그렇게 생각하는지 설명하세요.
- 로그 파일이나 에러 메시지를 포함하세요.

### 기능 제안

- 왜 이 기능이 유용한지 설명하세요.
- 제안하는 기능에 대한 단계별 설명을 제공하세요.
- 고려해본 대안이나 다른 솔루션이 있다면 언급하세요.

### 풀 리퀘스트 (Pull Request)

1. 저장소를 Fork 합니다.
2. 새로운 브랜치를 생성합니다 (`git checkout -b feature/amazing-feature`).
3. 변경 사항을 반영합니다.
4. 테스트가 통과하는지 확인합니다 (`pytest`).
5. 변경 사항을 커밋합니다 (`git commit -m 'feat: add some amazing feature'`).
6. 브랜치에 Push 합니다 (`git push origin feature/amazing-feature`).
7. Pull Request를 생성합니다.

## 코드 스타일

- 모든 함수에 타입 힌트(Type Hints)를 사용하세요.
- PEP 8 가이드라인을 준수하세요.
- 린팅과 포맷팅을 위해 `ruff`를 사용하세요.
- 새로운 모듈이나 복잡한 로직에는 명확한 독스트링(Docstring)을 작성하세요.

## 테스트

우리는 테스트를 위해 `pytest`를 사용합니다. 새로운 기능이나 버그 수정 시 반드시 관련 테스트 코드를 포함해 주세요.

```bash
pytest
```
