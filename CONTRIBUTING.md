# Contributing to Multi-LLM Gateway

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to Multi-LLM Gateway. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs

- Use a clear and descriptive title.
- Describe the exact steps which reproduce the problem.
- Explain which behavior you expected to see and why.
- Include logs or error messages.

### Suggesting Enhancements

- Explain why this enhancement would be useful.
- Provide a step-by-step description of the suggested enhancement.
- Mention any alternative solutions or features you've considered.

### Pull Requests

1. Fork the repository.
2. Create a new branch (`git checkout -b feature/amazing-feature`).
3. Make your changes.
4. Ensure the tests pass (`pytest`).
5. Commit your changes (`git commit -m 'feat: add some amazing feature'`).
6. Push to the branch (`git push origin feature/amazing-feature`).
7. Open a Pull Request.

## Code Style

- Use type hints for all functions.
- Follow PEP 8 guidelines.
- Use `ruff` for linting and formatting.
- Write clear docstrings for new modules or complex logic.

## Testing

We use `pytest` for testing. Please ensure that all new features or bug fixes are accompanied by tests.

```bash
pytest
```

## Questions?

Feel free to open an issue with the "question" label.

Happy coding! 🚀
